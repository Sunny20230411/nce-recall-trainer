# 课文导入与解析规范

版本：1.0.0
状态：项目实施基线

## 1. 固定原则

以后新增课程时，可以只提供英文原文。内容生产流程负责补齐中文翻译、逐句练习数据、单词信息、词性、句子结构和提示数据。

以下规则必须保持不变：

1. 原文顺序是唯一顺序来源，任何拆句都不得打乱或改写原文。
2. 默认练习单位是一条完整英文句子；过长复合句可按逗号或分号处的完整语义片段拆分，不把多个完整句子塞进同一道题。
3. 英文原句是答案基准；中文是帮助回忆的提示，不替代英文原句。
4. 词性和句子成分是两个独立层级。一个词可以是“名词”，同时所在短语可以是“主语”。
5. 句子结构必须在内容生产阶段生成并保存。答题页面只读取和展示，不在浏览器中临时猜测。
6. 自动生成结果必须经过数据校验；复杂句、倒装句、省略句和多义句进入人工抽检队列。
7. 人工修订优先级高于自动生成，再次批量生成时不得覆盖人工确认内容。
8. 每次内容生成都记录规则版本、生成方式和审核状态，保证后续可追溯。

## 2. 内容生产流程

```text
英文原文
→ 保留段落和对话顺序
→ 按句末标点拆成练习句
→ 中文翻译
→ 单词切分与标点归属
→ 音标、词性、语境词义
→ 句子结构与嵌套关系
→ 三层记忆提示
→ 自动完整性校验
→ 复杂句抽检/人工修订
→ 发布为课程版本
```

### 2.1 拆句

- 以 `. ! ?` 为主要边界。
- `Mr.`、`Mrs.`、`Dr.`、`a.m.`、`p.m.`、首字母缩写中的句点不能误拆。
- 引号中的完整句子可以独立成题，但保留引号和原始标点。
- 一段包含多句时拆成多题，`sourceParagraphOrder` 和 `sourceSentenceOrder` 用于恢复原顺序。
- 超过 14 个词的复合句可按逗号或分号拆为练习片段；英文和中文必须边界对齐、顺序不变，少于 3 个词的碎片要与相邻片段合并。
- 拆分片段使用原句稳定 ID 加 `-part-01`、`-part-02` 后缀，并保留 `parentSentenceId`，不得改写原文或在答题时临时拆分。
- 标题、说话人姓名和舞台说明不是答案正文，除非原文明确要求学习。

### 2.2 中文翻译

- 以当前语境自然表达为准，不逐词硬译。
- 人名、称呼、礼貌程度、问句和感叹语气必须保留。
- 中文提示只对应当前英文句，不合并相邻句的信息。
- `translationStatus` 依次为 `generated`、`reviewed`、`approved`。

### 2.3 单词与标点

- 一个输入格对应一个可输入英文词项。
- 句末和紧邻单词的标点存入该词项的 `punctuation`，不单独占输入格。
- 缩写形式如 `isn't`、`Here's` 作为一个词项。
- 每个词项保存显示文本、标准化文本、音标、词性、词性类别和当前语境中文义。
- `tokenIndex` 从 0 开始，必须连续且与页面输入格完全一致。

### 2.4 句子结构

- 每个结构组用 `[startToken, endToken]` 表示闭区间，索引从 0 开始。
- 父子结构通过 `parentGroupId` 表示；子范围必须完全位于父范围内。
- 同一父级下的结构组不得重叠。
- 顶层结构应覆盖全部词项；不能分析的独立表达使用 `independent_element`，不得静默遗漏。
- 结构标签支持主语、谓语、宾语、表语、补语、状语、呼语、独立成分、主句、并列句和各类从句。
- Lesson 3 已人工确认的数据属于人工覆盖层，自动生成不得替换。

结构类型全集以 [course-content.schema.json](../schemas/course-content.schema.json) 中的枚举为准。

### 2.5 三层提示

每句可保存三层提示：

1. `memory_note`：结合场景的一句短提示，不直接给答案。
2. `letter_shape`：保留每个词首字母，其余字母用 `_` 遮罩，保留空格和标点。
3. `answer_reveal`：完整答案和一句轻量记忆说明。

提示是内容数据，不应在每次点击时随机变化。

## 3. 统一 JSON 内容包

新内容使用 [course-content-template.v1.json](../content/course-content-template.v1.json) 的结构。内容包分三个阶段：

- `raw`：只要求课程元数据和英文原文，可交给内容生产流程继续处理。
- `enriched`：已包含中文、词项、结构和提示，可进入自动校验。
- `approved`：已通过自动校验和必要的人工复核，可以发布。

完整字段可以参考 [course-content-example.enriched.v1.json](../content/course-content-example.enriched.v1.json)。

稳定 ID 规则：

- `courseId`：课程包永久标识，例如 `nce-1`。
- `lessonId`：`{courseId}-lesson-{三位课号}`。
- `sentenceId`：`{lessonId}-sentence-{三位顺序}`。
- `tokenId`：`{sentenceId}-token-{三位索引}`。
- 人工修订时保留 ID；修改文字通过 `revision` 递增，不重新创建身份。

## 4. 自动校验门槛

发布前必须全部通过：

- 课程、课文、句子 ID 唯一。
- 课文和句子顺序为正整数且不重复。
- 英文原句不能为空。
- `enriched`/`approved` 内容必须有中文翻译。
- 词项索引从 0 连续增长，拼回后可对应原句。
- 结构索引不越界，父组存在，子组不越父组，同级不重叠。
- `approved` 内容必须有结构覆盖、规则版本和审核状态。
- 人工覆盖内容带 `analysisSource: "human"`，不能被生成脚本覆盖。

运行：

```bash
npm run validate:content -- content/course-content-template.v1.json
```

## 5. 数据库存储边界

当前 Demo 继续使用静态文件和浏览器本地学习记录。未来接入服务端时采用 [schema.postgres.sql](../database/schema.postgres.sql)：

- 内容域：课程包、课文、句子、词项、结构组、提示和内容版本。
- 学习域：练习会话、答题记录、句子掌握度和课文进度。
- 内容发布后按版本读取；用户历史答题记录关联具体内容版本，避免改课文后统计错位。

## 6. 后续新增课文的标准交付

每次用户只提供英文原文时，默认交付以下结果：

1. 已拆句且顺序稳定的课程内容包。
2. 每句中文翻译。
3. 每词音标、词性和语境义。
4. 每句结构分组和嵌套关系。
5. 三层提示。
6. 自动校验报告和需人工关注的复杂句列表。
7. 可直接加载到现有答题页面的静态数据文件。

这份文档是项目长期规则。以后扩展第二册、第三册或自定义课程，默认沿用本规范，除非产品明确修改规则版本。
