# 新概念课文默写训练阶段性总结与检查报告

- 检查日期：2026-07-15
- 检查范围：当前本地工作树中的产品功能、答题规则、内容规范、数据结构、代码维护性、桌面/移动端界面与发布配置
- 基线版本：`v1.0.0` 为此前本地稳定标记；本报告覆盖其后的阶段迭代
- 技术形态：原生 HTML、CSS、JavaScript 静态单页应用，Vercel 输出目录为 `dist`

## 1. 阶段结论

当前版本已经形成可演示、可连续练习、可保存本地学习记录的完整闭环：

1. 首页选择课程包。
2. 第一册课程列表选择具体课文。
3. 课文预览中查看英文原文和中文翻译。
4. 进入逐句中译英默写。
5. 使用键盘完成分词输入、切格、提交、纠错和下一题。
6. 答对后展示音标、词性、词义和句子结构。
7. 完成后生成成绩、错句和复习入口。
8. 浏览器刷新或关闭后仍可保留已完成的学习记录与兼容的未完成会话。

本阶段可以继续作为 Demo 上线。需要明确：学习记录目前只存于当前浏览器，数据库结构尚未接入线上服务；全量句子结构是按规则生成并固化的机器标注基线，不等同于语言专家逐句人工审校。

## 2. 已实现功能

### 2.1 页面层级

- 首页：选择新概念第一册、第二册或其他课程入口。
- 课程列表：展示 72 节奇数课正文，全部解锁，并显示学习进度、正确率和最近学习状态。
- 课文预览：双栏展示英文原文与中文翻译，提供开始练习入口。
- 练习页：逐句输入、即时校验、正确解析、显示答案、上一题/下一题和快捷操作。
- 结果页：展示得分、提交正确率、一次命中率、错句、用时、重听次数、错题复习、重练整课和下一课。

### 2.2 答题输入规则

- 进入题目后自动聚焦第一个单词格。
- 每个英文单词对应一个独立输入格，标点由标准答案保留，不单独占输入格。
- 普通输入状态下，空格逐格向后切换；最后一格按空格提交整句。
- 左右方向键只负责切换输入格，不承担提交或改写行为。
- 空输入格按退格键可退到上一格，连续退格可一直删除到第一格。
- 输入格宽度随当前输入内容增长；校验失败并清空错误词后恢复该格初始宽度。
- 首轮校验后若只剩一个错误格，重填该词并按空格可直接重新校验整句。
- 输入判断忽略大小写、句末标点和多余空格，但单词、数量和顺序必须正确。

### 2.3 错误与正确状态

- 校验失败时一次标出所有错误词的红色下划线。
- 错误词内容被清空，正确词保留。
- 焦点自动回到最左侧错误格，可立即重新输入。
- 错误状态不展开音标、词义或整句答案，不移动中文提示。
- 校验成功后停止编辑，单词与下划线进入正确状态。
- 答对后显示音标、词性、上下文词义和句子结构分组。
- 正确状态按 Enter 进入下一题，并自动聚焦第一格。

### 2.4 辅助操作

- 显示答案为固定占位模块，显示/隐藏不会挤压中文提示和输入区。
- 显示答案只控制答案模块，不隐藏中文提示。
- 播放发音使用浏览器语音能力。
- “掌握”和“生词”状态可参与本次统计和本地学习记录。
- 练习中返回课程列表会二次确认；确认退出后回滚到本次练习开始前的课程记录。
- 未确认退出时可继续原练习。
- 深色和浅色主题覆盖首页、课程卡片、练习状态、弹窗和结果页。

### 2.5 统计口径

- 提交正确率 = 正确提交次数 / 全部提交次数。
- 同一句多次提交错误会降低提交正确率。
- 一次命中率 = 首次提交即正确的句子数 / 本次答题句子数。
- “完美”表示首次答对，“很好”表示重试后答对，“跳过”单独统计。
- 错句列表记录最后一次错误输入和尝试次数。

## 3. 内容与解析数据

### 3.1 当前静态题库

- 第一册运行时共有 956 个句子实例。
- 标准化去重后共有 875 条静态句子结构数据。
- 结构覆盖缺失为 0，不支持的结构类型为 0。
- Lesson 3 的人工复核标注优先于生成数据。
- 其他课程使用预生成并固化的结构数据，答题过程中不会临时猜测。

### 3.2 内容生产规范

`docs/content-import-and-analysis-spec.md` 已定义后续新增课文的标准流程：

1. 导入英文原文。
2. 按原顺序拆为可练习单句。
3. 生成并审校中文翻译。
4. 生成单词、标准化文本、音标、词性和上下文词义。
5. 生成句子结构分组及父子层级。
6. 生成三层提示数据。
7. 通过 Schema 和业务规则校验后发布。

统一内容包使用 `schemas/course-content.schema.json`。模板和完整示例位于 `content/`，校验程序为 `scripts/validate-content-package.mjs`。

### 3.3 数据库边界

`database/schema.postgres.sql` 已给出课程、课文、句子、词元、结构分组、提示和用户学习记录的关系型设计。当前网页没有连接数据库，实际运行数据分为：

- 静态课程内容：随网页构建发布。
- 学习记录：`localStorage` 的 `nce_learning_records_v1`。
- 未完成会话：`localStorage` 的 `nce_practice_session_v3`。
- 导入课程：当前浏览器本地存储。
- 主题：当前浏览器本地存储。

上线数据库前应增加用户身份、服务端 API、数据迁移、权限校验和同步冲突策略。

## 4. 代码结构与维护性检查

### 4.1 已具备的工程基础

- Vercel 构建配置清晰：`npm run build`，输出 `dist`。
- `.gitignore` 已覆盖 `node_modules`、`.next`、`dist`、`.env` 和 `.env.local`。
- 内容 Schema、验证程序、生成程序和数据库草案已经从页面代码中独立出来。
- 结构数据在构建前生成，前端只读取静态结果，运行时行为可重复。
- 人工覆写优先级已明确，生成程序不会覆盖 Lesson 3 的已审校内容。

### 4.2 主要风险

#### P1：页面文件职责过多

`index.html` 约 7,848 行、102 个函数，同时承担样式、题库、视图、练习状态机、存储、统计和渲染。修改一个区域时容易影响其他状态，后续应优先拆成：

- `data/`：课程和语言内容加载。
- `practice/`：输入、校验、状态机和快捷键。
- `persistence/`：学习记录与会话迁移。
- `analysis/`：词元和结构渲染。
- `views/`：首页、列表、预览、练习和结果。
- `styles/`：主题、布局和组件状态。

#### P1：缺少自动化测试

目前构建脚本只复制静态文件，尚无单元测试、浏览器回归测试、代码检查和格式检查。最优先应覆盖：

- 单词标准化和答案比对。
- 空格、方向键、退格、Enter 的状态转换。
- 多轮错误后正确词保留与单错词快速重提。
- 显示答案的布局稳定性。
- 未完成退出的记录回滚。
- 提交正确率和一次命中率。
- 结构分组覆盖和嵌套合法性。

#### P1：课程数据存在多个来源

当前英文题库和部分中文覆写仍内嵌在 `index.html`，仓库同时保留 `nce1-data.js`、`nce1-zh-overrides.js` 和补充文件。生成程序还通过截取 `index.html` 中的变量读取题库。后续应收敛为单一 JSON 内容包，页面、生成器和校验器全部读取同一来源，避免翻译、句数和结构索引漂移。

#### P2：标准化逻辑重复

前端、结构生成器和内容验证器分别实现文本标准化。规则调整时存在键值不一致风险，应抽为共享模块，并给典型缩写、撇号和标点建立固定测试样例。

#### P2：版本标记未统一

Git 已有 `v1.0.0` 稳定标签，但 `package.json` 仍为 `0.1.0`。这不影响部署，但下一阶段发布前应统一应用版本、Git 标签和内容规则版本。

#### P2：机器结构标注仍需抽检

全量覆盖不等于全量语言审校。简单句已经适合演示，复杂从句、倒装、省略、呼语和歧义词性应按课程批次抽查，并使用 `reviewed`、`approved` 状态区分质量等级。
